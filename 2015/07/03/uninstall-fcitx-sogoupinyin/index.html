解决Ubuntu卸载fcitx后桌面无法登录的问题

发表于 2015-07-03 | 更新于 2016-04-14 | 分类于 Linux
问题描述：删除fcitx后无法登录到桌面环境，输入密码后闪屏一次，重新输入密码。问题原因：卸载fcitx没有同时删掉sogou拼音，残留文件导致无法登陆。

##问题描述
本来sougou拼音在linux里是一个非常良心的存在，却需依赖fcitx，在我使用期间经常出现假死状态，需要在其他界面刷新一下才能恢复。那种码字正爽的时候，突然不能切换中英文的感受我实在不能接受了。所以打算卸载fcitx，在IBus里使用Google拼音。

使用的卸载命令是：

$ sudo apt-get autoremove fcitx

然后启用IBus，在终端输入：

$ ibus-setup

提示IBus是关闭状态，是否开启，一路点是。然后提示在系统设置-语言设置里-选择IBus

一切准备完成，注销。再登录，然后K.O.了，登陆不了。

##解决方法
之后才想起来，卸载了fcitx，sogou拼音还在，导致登录后不能加载输入法。

所以完整删除sogou拼音即可。

###第一次尝试
删除/etc/X11/Xsession.d/目录下sogou拼音的配置文件

$ cd /etc/X11/Xsession.d
$ sudo ls

找到sogou拼音相关的一项，我的是72sogoupinyin

$ sudo rm -f 72sogoupinyin

然后再看/etc/X11/xinit/xinput.d目录，我再里面看到了fcitx，所以就删掉了

然后Ctrl+Alt+F7切换到桌面登录，还是不行。

###第二次尝试
在登录界面按Ctrl+Alt+F1，进入tty1字符界面
输入用户名和密码登录，然后执行下面命令：

$ sudo apt-get purge sogoupinyin

运行这条命令之后，会看到删了很多残留文件。

然后按Ctrl+Alt+F7回到桌面登录，再输入密码就能登录了。

sogou拼音的算是删干净了，但是fcitx的还有残留，所以我又删了一次
命令：

$ sudo apt-get purge fcitx

##卸载sogou拼音输入法的正确方法

上面是解决了问题，但是避免更多人遇到这个问题。
写下Ubuntu正确删除sogou拼音输入法的过程。

找到安装的搜狗输入法

$ sudo dpkg -l so*

卸载搜狗输入法

$ sudo apt-get purge sogoupinyin

在系统设置里语言支持里输入方式选择为IBus

卸载fcitx

$ sudo apt-get purge fcitx

卸载fcitx及相关配置

$ sudo apt-get autoremove

注销

如果状态栏的注销无效，就使用命令强制关闭：

$ sudo pkill Xorg

重新登录

##总结
花了一晚上时间解决这个问题，感觉不是很值。
目前以fcitx作为输入框架对中文还是支持最好的，有sogou拼音和google拼音等供选择。而且sogou拼音中文词库丰富，是最佳选择。对于sogou拼音，有时（尤其是每次在先复制-粘贴了英文后）不能通过ctrl+space或shift切换中英文调出中文输入法，出现机率非常之高，目前发现的解决方案是按capslk切换大小写后可以重新使用中文输入法，或者刷新一下页面，才能切换成中文输入。
以IBus做输入框架，有google拼音和其他几种拼音输入法可供选择，但是或多或少有点bug，而且词汇少。
我切换回IBus使用google拼音，只是打字的时候有些候选字中，你选择了某个字，但是实际输入的却是其他字。就是候选词翻页有问题，不能及时翻页。但其他的还算流畅。
我觉得过几天可能又要安装sogou拼音了，暂时先不装了，所以至于选那种看自己了。
