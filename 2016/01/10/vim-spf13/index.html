Vim的spf13配置与操作

发表于 2016-01-10 | 更新于 2016-05-15 | 分类于 Vim

想要把vim当成IDE用，又觉得配置麻烦，那就使用spf13,偷懒专属。在gvim里spf13表现更佳。

配置vim

插件地址:https://github.com/spf13/spf13-vim

安装：

关于Leader

<Leader> Vim默认的是\,spf13修改为,

配色修改

onedark 是Atom的默认配色。我的vim使用solarized各种问题，就改用onedark了。
color theme网站：http://vimcolors.com/
安装vim-gnome后,在gvim里solarized表现完美。

字体

sudo apt install fonts-inconsolata

个人喜欢Inconsolata字体,是最受欢迎的编程字体之一。

vim-indent-guides 缩进线

https://github.com/nathanaelkane/vim-indent-guides
用来优化展示缩进， 在GVim 中这个插件展示的很完美，可以自动识别背景色来展示。但是在终端环境中，只能显示dark/light两种模式，缩进看起来一块一块的，比较难看。
在.vimrc.local 中添加let g:indent_guides_auto_colors = 0关闭。 开启方式<Leader>ig

NERDTree

https://github.com/scrooloose/nerdtree
<C-E> 开启关闭NERDTree
<leader>e or <leader>nt 定位到当前打开文件的位置
:help NERDTree查看帮助

ctrlp

https://github.com/kien/ctrlp.vim

Vim中的文件查找器
:CtrlP or :CtrlP [starting-directory] 启动ctrlp，快捷键 <C-P>
:CtrlPBuffer or :CtrlPMRU 启动ctrlp并只在buffer/mru中进行查找
:CtrlPMixed 在 Files, Buffers 和 MRU 中进行混合查找

开启后的更多操作

surround

https://github.com/tpope/vim-surround
用来处理包围符号的插件，包括’ “ { } [ ]等
用法示例

如果光标在"foo bar"之间，可以使用cs"'转换成'foo bar'
*表示光标位置。:help surround查看帮助

nerdcommenter注释

https://github.com/scrooloose/nerdcommenter

<Leader>cc 注释一行
<Leader>cu 取消注释一行
<Leader>ca 切换注释模式// 或 /**/
<Leader>cm 以block的最简方式注释一段代码
<Leader>ci 以独立注释单行的方式 注释选中的每一行， 注释模式取决<Leader>ca
<Leader>cs 以block的常规方式注释一段代码
<Leader>cA 在行尾巴追加注释，注释模式取决于<Leader>ca

关于补全

spf13 提供了4种补全插件供我们选择(只能选择其中之一)
neocomplcache https://github.com/shougo/neocomplcache
YouCompleteMe https://github.com/Valloric/YouCompleteMe
neocomplete https://github.com/Shougo/neocomplete.vim

默认使用neocomplete，需要lua支持，默认的vim好像没有带。

vim --version | grep lua

自己编译安装，或者安装vim-nox,vim-gnome,vim-gtk都是可以的。
可以看到+lua就是支持了。

tabular对齐

https://github.com/godlygeek/tabular
可以对 = : , 做对齐操作

<Leader>a= 对=进行对齐操作
<Leader>a: 对:进行对齐操作
<Leader>a:: 对: 后面的内容做对其操作，而:靠前紧贴
<Leader>a, 对,进行对齐操作
