品胜 WPR001N 和 TP-Link WR703N 挂载U盘扩展磁盘空间

发表于 2017-02-14 | 更新于 2017-03-14 | 分类于 OpenWrt
扩展路由系统存储空间，WPR001N 和 WR703N 通用

前言

目前(2017.2), OpenWrt支持的最新版是15.05.1

新版升级了内核(3.18.23)，以前的老方式不可用。从网上找到了新的方法，操作起来更方便了

准备工作

准备一个U盘，我选择的是8G的，刚好有闲置的，给U盘分了3个区，2G ext4 作挂在系统, 128M swap, 剩余的ext4 作数据区 （分区工具推荐Linux下的GParted）

以下命令基本都是ssh登录到路由后操作的

ssh root@192.168.1.1

安装依赖包

挂在U盘需要以下三个包
kmod-usb-storage block-mount kmod-fs-ext4

如果路由能联网，执行下面命令安装

opkg update
opkg install kmod-usb-storage block-mount kmod-fs-ext4

路由不能联网的去官方库里下载相应的包，使用scp命令复制到路由安装
官方库:
https://downloads.openwrt.org/chaos_calmer/15.05.1/ar71xx/generic/packages/base/

这三个包还依赖以下包，下载一并安装
kmod-crypto-hash
kmod-lib-crc16
kmod-scsi-core

我就是手动安装的，比如我安装了下面几个包
block-mount_2016-01-10-96415afecef35766332067f4205ef3b2c7561d21_ar71xx.ipk
kmod-crypto-hash_3.18.23-1_ar71xx.ipk
kmod-fs-ext4_3.18.23-1_ar71xx.ipk
kmod-lib-crc16_3.18.23-1_ar71xx.ipk
kmod-scsi-core_3.18.23-1_ar71xx.ipk
kmod-usb-storage_3.18.23-1_ar71xx.ipk

检测U盘

将U盘插上，执行block info命令，看下U盘是否已被识别, 会有sda1, sda2, sda3，分别对应ext4分区, swap分区, ext4分区

转移数据

mount /dev/sda1 /mnt  
mkdir /tmp/root  
mount --bind / /tmp/root  
tar -C /tmp/root -cvf - . | tar -C /mnt -xvf -  
sync   
umount /tmp/root

设置swap

mkswap /dev/sda2  
swapon /dev/sda2

把当前挂在状态写入到fstab文件:

block detect > /etc/config/fstab

修改/etc/config/fstab文件

将sda1作为根分区, sda2 作为swap， sda3作为root目录:
/mnt/sda1改为/
/mnt/sda3改为/root
将config mount 和config swap的 option enabled都改为1

然后重启完成

重启后登录路由器执行df -h查看磁盘信息
